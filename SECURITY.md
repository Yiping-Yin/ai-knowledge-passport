# Security Policy

## Supported Scope

当前仓库重点关注以下安全问题：

- 本地知识数据泄露风险
- 未经授权的外部发送或导出
- 文件导入、解析、转写、OCR 等链路中的命令执行或路径安全问题
- 备份、恢复与对象存储中的敏感数据暴露
- 提示注入导致的越权检索、越权编译或错误授权

## Reporting a Vulnerability

请不要在公开 issue 中直接披露安全漏洞细节。

建议流程：

1. 先通过私下渠道联系维护者。
2. 说明影响范围、复现步骤、潜在风险和建议修复方向。
3. 在获得确认前，不要公开完整 exploit 或敏感细节。

如果 GitHub 私有漏洞报告已开启，优先使用该渠道。

## What to Include

请尽量包含：

- 受影响模块或路径
- 复现步骤
- 触发条件
- 实际影响
- 可能的修复建议

## Response Goal

项目会尽量在确认后快速完成：

- 漏洞分级
- 修复方案确认
- 补丁发布
- 必要时的公开说明
